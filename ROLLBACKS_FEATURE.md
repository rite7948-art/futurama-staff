# 📦 Загрузка откатов (больших файлов) в отчеты

## Что изменилось

Теперь в систему отчетов добавлена возможность загружать **файлы откатов** (архивы) большого размера - до **500 МБ**.

## Основные изменения

### 1. **Увеличены лимиты PHP** (`php.ini`)
- `upload_max_filesize`: 500 MB (было 2 MB)
- `post_max_size`: 500 MB (было 8 MB)
- `memory_limit`: 512 MB

### 2. **Обновлена база данных**
Добавлены новые поля в таблицу `reports`:
- `rollback_file` - путь к файлу отката
- `file_type` - расширение архива (zip, rar, 7z и т.д.)
- `file_size` - размер файла в байтах

### 3. **Поддерживаемые форматы откатов**
✅ ZIP
✅ RAR
✅ 7Z
✅ TAR
✅ GZ (GZIP)
✅ TAR.GZ (TGZ)
✅ TAR.BZ2
✅ И другие архивные форматы

### 4. **Новый интерфейс в форме отчетов**
- Добавлено поле **"Файл отката (опционально)"**
- Поддержка drag-and-drop для архивов
- Валидация типа файла и размера
- Отображение имени и размера файла перед загрузкой

### 5. **Отображение откатов в списке**
- В каждом отчете теперь видна кнопка для скачивания отката (если он загружен)
- Показывается тип архива и его размер
- Кнопка имеет фиолетовый цвет для отличия от скриншотов

## Как использовать

### Загрузка отката при создании отчета:

1. Заполните основные поля (ник, Discord ID)
2. Загрузите скриншот собеседования (обязательно)
3. **Опционально**: загрузите файл отката (архив) в нижнее поле
4. Нажмите "Отправить отчет"

### Где сохраняются файлы

- **Скриншоты**: `/uploads/`
- **Откаты**: `/rollbacks/`

## Технические детали

### Папки
```
project/
├── uploads/          # Скриншоты отчетов (обычные размеры)
├── rollbacks/        # Файлы откатов (до 500 MB)
└── php.ini          # Конфиг для больших файлов
```

### Docker
Файл `Dockerfile` обновлен:
- Копирует `php.ini` для больших лимитов
- Создает папку `/rollbacks/` с правами 777

## Ограничения и безопасность

- **Максимальный размер**: 500 MB на файл
- **Время ожидания**: 300 секунд (5 минут) для загрузки
- **Валидация**: Только разрешенные расширения архивов
- **Локальное хранилище**: Все файлы сохраняются локально на сервере

## Возможные проблемы и решения

### Проблема: "Файл слишком большой"
**Решение**: Убедитесь, что:
- Файл не превышает 500 MB
- На сервере достаточно свободного места на диске
- В `php.ini` правильно установлены лимиты

### Проблема: "Неподдерживаемый формат"
**Решение**: Используйте один из поддерживаемых формантов архивов (ZIP, RAR, 7Z, TAR.GZ и т.д.)

### Проблема: Медленная загрузка
**Решение**: 
- Сжимайте файлы перед загрузкой
- Загружайте по частям, если возможно
- Используйте более быстрое интернет-соединение

## Примечания для администраторов

- Папка `rollbacks/` должна быть доступна для записи
- Рекомендуется регулярно очищать старые откаты для экономии места
- Следите за свободным местом на диске
- При необходимости увеличения лимитов, отредактируйте `php.ini`
